# Definition of Done

## Story Level

A user story is **Done** when:

- [ ] All acceptance criteria met and verified
- [ ] All tasks completed
- [ ] Code reviewed (self-review for sub-agents)
- [ ] Story integrated into running software
- [ ] PO has tested the increment containing this story
- [ ] No critical PO issues blocking acceptance
- [ ] No blocking bugs
- [ ] PO has accepted the story in Sprint Review

## Task Level

A task is **Done** when:

- [ ] Implementation complete per task description
- [ ] Code compiles/builds without errors
- [ ] No console errors or warnings
- [ ] Self-tested locally
- [ ] Code follows project conventions
- [ ] Files committed with proper message format

## Role-Specific Checklists

### Frontend Tasks
- [ ] Component renders correctly
- [ ] Responsive across breakpoints (mobile/tablet/desktop)
- [ ] Keyboard navigation works
- [ ] Loading/error states handled
- [ ] No TypeScript errors
- [ ] Tailwind classes are semantic

### Backend Tasks
- [ ] Function executes without errors
- [ ] Input validation implemented
- [ ] Error handling with meaningful messages
- [ ] Type-safe (TypeScript)
- [ ] Tested with sample data
- [ ] No security vulnerabilities (auth checks in place)

### Full-Stack Tasks
- [ ] Backend functions working
- [ ] Frontend components working
- [ ] End-to-end integration verified
- [ ] Auth/permissions correct
- [ ] Real-time updates working (if applicable)

## Sprint Level

A sprint is **Done** when:

- [ ] Sprint Goal achieved (or deviation explained)
- [ ] All committed stories Done or explicitly moved
- [ ] Sprint Review conducted with PO
- [ ] Sprint Retrospective completed
- [ ] Sprint Summary documented
- [ ] Backlog updated for next sprint

## Quality Gates

### Before Sprint Planning
- [ ] Stories are "Ready" (clear criteria, estimated)
- [ ] Tech stack confirmed in config.md
- [ ] No unresolved blockers from previous sprint

### Before Sprint Review
- [ ] All "Done" stories have working software
- [ ] Demo environment prepared
- [ ] Known issues documented

### Before Retrospective
- [ ] Sprint metrics calculated
- [ ] Team has reflected on the sprint
- [ ] Action items from previous retro reviewed
